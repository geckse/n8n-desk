# n8n-desk Local Agent — Revalidation Findings

> Result of a 7-dimension multi-agent audit (backend parity, MCP tool surface, sandbox/security, system prompts, IPC/streaming lifecycle, renderer UX, daily-task product gaps). Every finding was adversarially verified: critical/high findings faced 3 independent skeptics (2-of-3 required to survive), medium/low faced 1. The audit ran to completion including the completeness sweep. Findings only — fix plan omitted per current scope.

## Summary

**66 confirmed findings** — 4 critical, 21 high, 28 medium, 13 low. **3 findings were refuted** by the verification pass (listed at the end — do not act on them).

The `votes` on each finding show how many skeptics upheld it (e.g. `3/3` = unanimous). `correctedSeverity` reflects any severity the verifiers adjusted from the original report.

### The load-bearing failures

The local agent is supposed to keep two backends at feature parity (CLAUDE.md rule): **Deep Agents SDK** (OpenAI/Ollama) and **Claude Agent SDK** (Anthropic). The audit found parity broken at the foundation, and the human-in-the-loop approval contract non-functional on *both* backends:

1. **Claude SDK approval gate never fires** — checks bare tool names against the SDK's `mcp__n8n__*` namespaced names, so `execute_workflow`/`create`/`publish`/`archive` run **without user approval**.
2. **Deep Agents runner is non-functional** — normalizes `streamEvents`-shaped events from an `agent.stream()` call that never emits them, and never resumes LangGraph interrupts. That backend streams nothing and never runs approved tools.
3. **Deep Agents built-in file tools bypass the sandbox** — `~/.n8n-desk` mounts as read-write ignoring `ro` mode; `llm.json`/`auth.json` are readable and the read-only mount is writable.
4. **MCP tool schemas drifted from the real n8n server** — `search_nodes`/`get_suggested_nodes` fail server validation every call, `execute_workflow` silently drops all inputs; the server-mandated `get_sdk_reference` is missing and the server's ~30-tool surface is unreachable via dynamic discovery.

### Newly-covered areas from the completeness sweep

The sweep flagged three areas the initial dimensions under-covered; findings from them are folded into the list below where confirmed:
- Electron packaging/build of agent dependencies (esbuild bundling + asar + Claude Code subprocess spawn)
- LLM provider feature loss for OpenAI/Ollama vs Anthropic (prompt caching, tool-calling, context-window/summarization, generation params)
- Internationalization of agent-facing renderer UI (hardcoded English in agent chat/tool/approval components)

### ~~Case-insensitive filesystem (macOS) defeats the sensitive-file deny-list: reading LLM.json returns llm.json API keys~~

### ~~Stateless per-call MCP connections add round trips per tool call, never terminate server sessions, and discard the server's initialize instructions~~
- **File:** `electron/mcp-client.ts`
- **Why refuted:** The finding's central mechanism is refuted by the n8n-master ground truth.
